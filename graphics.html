Cascaded shadowmapping

In this project I implemented cascaded shadowmapping for main directional light of the scene on my engine project. In cascaded shadowmapping view frustum is split into multiple parts (cascades) and a shadow texture is assigned to each of them. Cascade that is nearest to the camera is smallest, making it possible to render high quality shadow textures near the viewer. Size of the cascades grow exponentially further away from the viewer up until the far plane. For each cascade, orthographic projection is perfectly fitted around the split view frustum for highest possible shadow quality for each cascade. Microsoft guide for cascaded shadowmaps was used in the implementation.

Constrained dynamics

Physical simulation with constraint forces. Constraints for certain properties such as particle-to-particle distance can be set. Physical simulation is run, constraint derivatives are calculated which are then used to calculate a force, which enforces the constraints. Implemented method described by Andrew Witkin.

Inverse kinematics (Jacobian pseudoinverse)

Inverse kinematics with automatic differentation. Given a target position for a node of a skeleton, the method calculates the "optimal" step taken in joint rotations towards the target. To reach the target, multiple smalls steps are taken. For each step, the Jacobian of target position for each joint angle is calculated with forward automatic differentation (operator overload). As the resulting Jacobian matrix is rarely even, pseudo inverse is used to solve for the delta rotations for each joint. Moore-Penrose pseudoinverse can be computed from singular value decomposition. In this implementation joints can be locked in place to limit body movement.
